The baseline initialization phase has successfully concluded. The network has achieved complete operational equilibrium, and the protocol is officially open to global, fully decentralized consensus participation.

## 🛠 Core Technical Breakthroughs

Q-BTC is not a fork; it is a fundamental architectural redesign optimized for the post-quantum era:

### 1. Actor-Driven State Machine & Atomic Rollback (`utxo.rs`, `main.rs`)
Eradicates Mutex locks in the UTXO set. Introduces an OS-exclusive `UtxoActor` model, converting all state changes into MPSC message streams for physical serial isolation. Coupled with RocksDB's absolute synchronous batching (`set_sync(true)`), it achieves O(1) atomic physical rollback for deep Nakamoto reorganizations, granting absolute immunity to torn-states caused by power failures.

### 2. Asymmetric Cold Storage & PQ-SegWit Pruning (`storage.rs`, `transaction.rs`)
Counters the massive ~3.3 KB ML-DSA-65 signature bloat. Core state (`PREFIX_CORE_BLOCK`) and Witness data (`PREFIX_WITNESS`) are physically bifurcated in the KV-store. Via the `prune_historical_witnesses` mechanism, edge nodes can safely erase historical lattice signatures from physical sectors post-consensus, violently compressing long-term ledger expansion to classical Bitcoin thresholds.

### 3. Topological DAG Mempool & Tombstone Graveyard (`blind_box.rs`)
Defends against high-frequency Replay Floods and complex CPFP attacks. The mempool operates as a strict Directed Acyclic Graph (DAG) using `IndexMap`. Upon breaching the 1.2B Weight Units physical redline, a `BTreeSet` executes O(1) cascade evictions of low-fee branches. Invalidated malicious hashes are permanently marked in an immutable Tombstone filter, systemically immunizing the P2P network against rebroadcasted zombie transactions.

### 4. Compact Blocks (Q-BIP-152) & Unified View Sync (`p2p.rs`, `block.rs`, `storage.rs`)
Natively integrates Q-BIP-152 compact routing. Nodes broadcast only 64-bit salted `short_ids`, eliminating lattice-induced bandwidth congestion. During Initial Block Download (IBD), an isolated `staged_blocks` unified view is constructed in memory to mathematically pinpoint the Lowest Common Ancestor (LCA). It strictly prevents persistent database pollution prior to batch writing, entirely neutralizing disk-exhaustion attacks from adversarial forks.

### 5. Q-SigCache & Par-Iter Validation (`main.rs`, `utxo.rs`)
Shatters the single-thread execution limits of the Satoshi architecture. Leverages `rayon::into_par_iter()` in the assembly pipeline to concurrently shred ML-DSA matrix multiplications across heterogeneous CPU cores. Transactions verified in the mempool are cached in `verified_tx_cache`, achieving O(1) block finalization and preventing CPU stalling from the ground up.

### 6. Layer-1 Delayed Recovery Protocol (`transaction.rs`, `utxo.rs`)
Native disaster recovery for pure physical environments without Turing-complete smart contracts. Embeds `RecoveryInfo` natively within the Layer-1 `TxOut`. Utilizes hash commitments (`recovery_hash`) to hide backup keys, enforced by a consensus time-lock. This preserves a 30-day absolute veto window for the main key holder.

### 7. Monolithic Adherence & Future ZK-STARKs Roadmap (L2) (`consensus/`)
Rejects insecure sharding or cross-chain compromises, fiercely defending the Monolithic architecture baseline. While currently operating strictly as a robust Layer-1 basechain, the consensus module lays the architectural groundwork for a future Layer-2 evolution integrating ZK-STARKs. In its final roadmap vision, edge nodes will verify O(polylog N) complexity mathematical proofs rather than downloading massive historical ledgers, cementing the foundation for a true Stateless Verification paradigm.

## ⚙️ Building from Source

### Prerequisites
- Rust toolchain (`cargo` 1.75+)
- CMake & Clang (for RocksDB compilation)

### Quickstart
Clone the repository and build the optimized release binary:

    git clone https://github.com/Q-Jack-core/quantum-btc.git
    cd quantum-btc
    cargo build --release

The node binary will be available at `./target/release/qbtc-core`.
